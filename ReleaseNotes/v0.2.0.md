# codex-opero v0.2.0

## English

This release updates Antigravity quota reporting for the current Antigravity/agy quota service.

### Changes

- Added support for the group-level `RetrieveUserQuotaSummary` response used by Antigravity 2.1.4 in the current agy 1.0.9 environment.
- Antigravity details now show both `Weekly Limit` and `Five Hour Limit` for the Gemini and Claude/GPT model groups.
- Antigravity detail rows use the compact Codex-style labels `[5h]` and `[7d]`, with the five-hour window first.
- Updated local HTTP listener discovery for the new Antigravity service layout.
- Kept the previous per-model `GetAvailableModels` endpoint as a compatibility fallback.
- Added ad-hoc signing after app resources are copied so packaged app bundles pass local code-signature verification.
- Verified that missed weekly update windows are checked immediately on the next app launch, and hardened GitHub requests with a timeout, explicit API version, and cancellation-safe scheduling.

### Requirement

Keep the Antigravity app running and signed in while `codex-opero` reads its quota. The app does not launch `agy` automatically.

## 한국어

이번 릴리즈는 최신 Antigravity/agy quota service에 맞춰 Antigravity 사용량 표시 방식을 업데이트합니다.

### 변경 사항

- 현재 agy 1.0.9 환경의 Antigravity 2.1.4에서 사용하는 그룹 단위 `RetrieveUserQuotaSummary` 응답을 지원합니다.
- Antigravity 상세 영역에서 Gemini 및 Claude/GPT 모델 그룹의 `Weekly Limit`과 `Five Hour Limit`을 모두 표시합니다.
- Antigravity 상세 행을 Codex와 같은 축약 형식인 `[5h]`, `[7d]` 순서로 표시합니다.
- 변경된 Antigravity 로컬 service 구조에 맞춰 HTTP listener 탐색 방식을 수정했습니다.
- 이전 모델별 `GetAvailableModels` endpoint는 호환 fallback으로 유지합니다.
- 앱 리소스 복사 후 ad-hoc 서명을 적용해 패키징된 앱 번들이 로컬 코드 서명 검증을 통과하도록 수정했습니다.
- 주간 업데이트 확인 시점을 놓친 경우 다음 앱 실행 때 즉시 확인하도록 검증하고, GitHub 요청의 제한시간·API 버전 및 작업 취소 처리를 보강했습니다.

### 사용 조건

`codex-opero`가 quota를 읽는 동안 Antigravity 앱이 실행 중이고 로그인되어 있어야 합니다. 앱은 `agy`를 자동 실행하지 않습니다.
