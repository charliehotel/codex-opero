# Antigravity 상단 쿼터 요약 규칙

## 목표

Antigravity 메뉴 상단의 두 숫자가 각 모델 그룹의 실제 사용 가능 여부를 정확히 반영하도록 한다. 평소에는 먼저 소진되는 5시간 쿼터를 보여주되, 주간 쿼터를 모두 소진한 경우에는 5시간 쿼터가 남아 있거나 먼저 리셋되더라도 `0%`를 표시한다.

## 표시 규칙

각 모델 그룹의 상단 요약 값은 다음 순서로 결정한다.

1. 주간(`[7d]`) 잔여량이 `0%`이면 `0%`를 표시한다.
2. 주간 잔여량이 `0%`보다 크면 5시간(`[5h]`) 잔여량을 표시한다.

두 구간 중 항상 작은 값을 선택하지는 않는다. 예를 들어 `[5h] 100%`, `[7d] 97%`이면 상단에는 `100%`를 표시한다. `[5h] 69%`, `[7d] 0%`이면 상단에는 `0%`를 표시한다.

## 적용 범위

- 첫 번째 숫자에는 `Gemini Models` 그룹의 규칙 적용 결과를 표시한다.
- 두 번째 숫자에는 `Claude and GPT models` 그룹의 규칙 적용 결과를 표시한다.
- 상세 영역의 `[5h]`, `[7d]` 행과 원본 사용량은 변경하지 않는다.
- 이전 Antigravity 버전을 위한 `GetAvailableModels` 호환 경로는 기존 동작을 유지한다. 이 경로에는 별도의 주간 구간이 없으므로 현재 대표값을 그대로 사용한다.
- 독립 Gemini provider 제거와 함께 배포하며 앱 버전을 `0.2.1`로 올린다.

## 구현 방식

`RetrieveUserQuotaSummary` 응답을 `ProviderQuota`로 변환할 때 각 그룹에서 `[5h]`와 `[7d]` 구간을 찾는다. 상단 요약용 `QuotaWindow`는 5시간 구간을 기반으로 만들되, 주간 잔여량이 `0%`이면 사용량을 `100%`로 바꿔 잔여량이 `0%`가 되도록 한다. 상세 그룹에는 서버가 제공한 원본 구간을 그대로 보존한다.

## 검증

- `[5h] 69%`, `[7d] 0%`인 그룹의 상단 값이 `0%`인지 확인한다.
- `[5h] 100%`, `[7d] 97%`인 그룹의 상단 값은 `100%`로 유지되는지 확인한다.
- 상세 `[5h]`, `[7d]` 값과 순서가 바뀌지 않는지 확인한다.
- 전체 Swift 테스트를 실행하고 0.2.1 앱을 다시 패키징한다.
- 실행 중인 Antigravity의 실제 쿼터와 메뉴 표시를 비교한다.
