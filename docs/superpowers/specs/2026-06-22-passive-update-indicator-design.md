# 수동 방해 없는 업데이트 표시 설계

## 상태

- 작성일: 2026-06-22
- 사용자 승인: 완료
- 대상 버전: 0.2.1 이후

## 배경

현재 `codex-opero`는 마지막 성공 확인으로부터 약 일주일마다 GitHub Releases를 조회한다. 새 버전을 발견하면 모달 `NSAlert`를 띄우고, 사용자가 동의하면 Releases 페이지를 연다. 메뉴 하단에는 설치된 버전만 `v0.2.1` 형식으로 표시한다.

모달은 앱 사용 흐름을 막고, 거절하면 새 버전 정보가 메뉴에 남지 않는다. 새 설계는 모달을 제거하고 메뉴 하단의 버전 표시를 지속적인 업데이트 링크로 바꾼다.

## 목표

- 최신 상태에서는 현재 설치 버전을 조용히 표시한다.
- 새 안정 버전이 있으면 현재 버전과 최신 버전을 함께 표시한다.
- 업데이트 표시는 눈에 띄되 메뉴 사용을 방해하지 않는다.
- 사용자가 업데이트 표시를 클릭하면 해당 GitHub Release 페이지를 연다.
- 앱이나 Mac이 꺼져 확인 시점을 놓쳐도 다음 실행 때 즉시 확인한다.
- 네트워크 오류가 발생해도 이미 발견한 업데이트 상태는 잃지 않는다.
- macOS의 동작 줄이기 설정과 VoiceOver를 지원한다.

## 비목표

- 앱 내부 자동 다운로드 또는 자동 설치
- Sparkle 등 별도 업데이트 프레임워크 도입
- prerelease 또는 draft 릴리즈 안내
- 메뉴 막대 아이콘 자체에 업데이트 배지 추가
- 업데이트 표시를 사용자가 숨기거나 다시 알리지 않도록 하는 설정

## 결정 사항

### 기존 기능에서 유지할 부분

- GitHub `releases/latest` API 요청
- `AppVersion` 기반 의미 버전 비교
- draft와 prerelease 제외
- 요청 제한시간과 GitHub API 헤더
- 실패 후 재시도와 앱 종료 시 작업 취소
- 현재 번들 버전 판독 및 소스 실행용 `Resources/Info.plist` fallback

### 제거할 부분

- 새 버전 발견 시 표시하는 `NSAlert`
- `lastPromptedVersion`, `lastPromptedAt` 저장값과 중복 팝업 억제 정책
- 일주일 확인 주기

기존 UserDefaults에 남아 있는 팝업 관련 키는 더 이상 읽지 않는다. 호환성 문제는 없으므로 별도 마이그레이션은 필요하지 않다.

## 아키텍처

### UpdateChecker

기존 `UpdateChecker`를 폐기하지 않고 관찰 가능한 업데이트 상태 제공자로 확장한다. 네트워크 조회와 스케줄링 책임은 그대로 유지하고, 모달 표시 책임만 제거한다.

외부에 제공하는 상태는 다음 두 경우로 제한한다.

- `current`: 설치된 버전이 최신이거나 최신 버전을 알 수 없음
- `available`: 현재 버전, 최신 버전, 이동할 Release URL

`UpdateChecker`는 메인 액터에서 상태를 갱신한다. SwiftUI `ContentView`는 동일한 `UpdateChecker.shared` 인스턴스를 주입받아 상태 변화에 따라 하단 표시를 다시 그린다. `QuotaStore`에는 업데이트 상태를 넣지 않는다. 사용량 조회와 앱 업데이트 확인은 서로 다른 책임이기 때문이다.

### GitHub 응답

`LatestRelease`에 GitHub API의 `html_url`을 추가한다. 정상적인 HTTP(S) URL이면 해당 릴리즈 페이지를 사용하고, 값이 없거나 사용할 수 없으면 기존 Releases 목록 URL로 대체한다.

## 확인 주기와 상태 전이

`UpdateCheckPolicy.checkInterval`은 24시간으로 변경한다. 기준은 달력 날짜가 아니라 마지막 성공 확인 시각으로부터 흐른 24시간이다.

1. 앱 시작 시 현재 번들 버전을 읽는다.
2. 저장된 최신 버전이 현재 버전보다 높으면 네트워크 조회 전에도 `available` 상태를 복원한다.
3. 마지막 성공 확인 시각이 없거나 24시간 이상 지났으면 즉시 GitHub를 조회한다.
4. 실행 중에는 다음 확인 시각까지 기다린 뒤 다시 조회한다.
5. 성공 응답에서 더 높은 안정 버전을 발견하면 버전과 URL을 저장하고 `available`로 바꾼다.
6. 성공 응답에서 현재 버전이 최신이면 저장된 업데이트 정보를 지우고 `current`로 바꾼다.
7. 네트워크 또는 디코딩 오류가 발생하면 마지막 성공 확인 시각을 갱신하지 않고 한 시간 뒤 재시도한다.
8. 오류가 발생해도 복원되었거나 이전에 발견한 `available` 상태는 유지한다.
9. 사용자가 새 버전을 설치한 뒤 앱을 실행하면 저장된 최신 버전이 현재 버전보다 높지 않으므로 캐시를 지우고 `current`로 돌아간다.

저장 키는 다음으로 구성한다.

- 마지막 성공 확인 시각
- 발견한 최신 버전
- 발견한 릴리즈 URL

저장된 버전 문자열이나 URL을 해석할 수 없으면 해당 캐시는 버리고 `current`로 시작한다.

## 메뉴 UI

### 최신 상태

메뉴 하단 `Quit` 버튼 오른쪽에 현재처럼 흐린 `v0.2.1` 텍스트를 표시한다. 클릭 동작은 없다.

### 업데이트 가능 상태

- 표시: `v0.2.1 → v0.2.2`
- 형태: borderless SwiftUI 버튼
- 색상: 시스템 accent color
- 애니메이션: 완전히 숨기지 않고 불투명도가 약 `1.0`과 `0.45` 사이에서 부드럽게 왕복
- 주기: 한 번의 밝아짐·흐려짐 전체 주기 약 1.6초
- 클릭: 저장된 해당 Release URL을 기본 브라우저로 연다.

메뉴를 닫아도 업데이트 상태는 유지한다. 애니메이션은 메뉴가 화면에 표시되는 동안에만 의미가 있으며, 업데이트를 클릭해도 설치 확인 전까지 표시는 유지한다.

### 접근성

- `accessibilityReduceMotion`이 켜져 있으면 펄스 애니메이션을 적용하지 않는다.
- VoiceOver 레이블: `v0.2.2 업데이트 가능, GitHub Releases 열기`
- 시스템 accent color와 표준 버튼 포커스 동작을 사용한다.
- 텍스트가 완전히 사라지는 점멸 효과는 사용하지 않는다.

## 오류 처리

- GitHub 조회 실패는 사용자에게 팝업으로 알리지 않는다.
- 아직 업데이트를 발견하지 못한 상태에서 실패하면 현재 버전 표시를 유지한다.
- 이미 업데이트를 발견한 뒤 실패하면 저장된 버전과 링크를 계속 표시한다.
- 현재 번들 버전을 읽지 못하면 업데이트 비교를 수행하지 않고 메뉴 하단의 버전 영역도 표시하지 않는다.
- 브라우저 열기에 실패해도 앱 상태나 저장된 업데이트 정보는 변경하지 않는다.

## 테스트

### 단위 테스트

- 마지막 성공 확인으로부터 24시간 전에는 확인하지 않는다.
- 정확히 24시간이 지나면 확인한다.
- 확인 시점을 놓친 뒤 앱을 실행하면 즉시 확인한다.
- 조회 실패 후 한 시간 재시도 간격을 사용한다.
- 새 안정 버전을 `available` 상태로 변환한다.
- 동일하거나 낮은 버전, draft, prerelease는 업데이트로 처리하지 않는다.
- 저장된 더 높은 버전과 URL을 앱 시작 시 복원한다.
- 저장된 버전이 현재 버전과 같거나 낮으면 캐시를 삭제한다.
- 잘못된 저장값은 안전하게 무시한다.
- 조회 실패 시 기존 `available` 상태를 유지한다.
- `html_url`이 없거나 잘못됐을 때 Releases 목록 URL을 사용한다.
- 버전 표시 문자열이 `v0.2.1 → v0.2.2` 형식인지 확인한다.

### 실제 앱 검증

- 최신 상태에서 기존 `v0.2.1` 표시와 레이아웃이 유지되는지 확인한다.
- 테스트용 새 버전 상태에서 파란 펄스 버튼과 정확한 버전 문자열을 확인한다.
- 버튼 클릭 시 올바른 GitHub URL이 열리는지 확인한다.
- 동작 줄이기 설정에서 애니메이션이 정지하는지 확인한다.
- VoiceOver 접근성 레이블과 키보드 포커스를 확인한다.
- 메뉴를 닫았다 다시 열어도 업데이트 상태가 유지되는지 확인한다.

## 문서 변경

- README의 업데이트 확인 주기를 일주일에서 24시간으로 수정한다.
- 기존 팝업 설명을 메뉴 하단 업데이트 링크 설명으로 교체한다.
- 해당 기능이 포함되는 릴리즈 노트에 수동 방해 없는 업데이트 표시를 기록한다.

## 완료 조건

- 업데이트 팝업 코드와 중복 팝업 저장 정책이 제거된다.
- 새 버전 상태가 메뉴 하단에 지속적으로 표시된다.
- 24시간 확인 및 한 시간 오류 재시도 정책이 테스트로 고정된다.
- 캐시 복원과 설치 후 자동 해제가 테스트로 고정된다.
- 실제 패키징 앱에서 펄스, 클릭, 동작 줄이기, 접근성을 검증한다.
